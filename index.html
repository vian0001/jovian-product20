TOPOLOGI JARINGAN

Tampilan topologi di gns3

Topologi jaringan yang digunakan untuk implementasi DNS di MikroTik terdiri dari komponen-komponen berikut:

Router MikroTik berfungsi sebagai DNS server dan DNS cache untuk seluruh jaringan lokal. Semua klien dalam jaringan akan menggunakan router sebagai DNS resolver utama.

Berikut adalah komponen-komponen dalam topologi jaringan DNS:

1. MikroTik Router (192.168.88.1)
# Berfungsi sebagai DNS server dan DNS cache
2. Internet/ISP ‚Üí Sumber DNS Upstream
# Menyediakan resolusi DNS untuk domain publik (8.8.8.8, 1.1.1.1)
3. Jaringan LAN (192.168.88.0/24)
# Perangkat klien yang menggunakan layanan DNS MikroTik
4. Server Internal (192.168.88.254)
# Server internal yang di-resolve melalui Static DNS

Dalam topologi ini, MikroTik Router menangani semua permintaan DNS dari jaringan lokal. Untuk domain publik (seperti google.com), router akan meneruskan permintaan ke server DNS upstream (seperti 8.8.8.8). Untuk domain internal (seperti server.local), router akan menggunakan entri static DNS yang telah dikonfigurasi.

Router MikroTik harus dikonfigurasi dengan benar untuk membatasi akses DNS hanya untuk jaringan internal. Ini mencegah router digunakan sebagai open DNS resolver yang dapat dimanfaatkan untuk serangan DNS amplification.

Dengan menggunakan topologi ini, kita dapat mengoptimalkan kinerja DNS dengan memanfaatkan fitur static DNS dan DNS cache yang ditawarkan oleh RouterOS.

# Pengertian Gateway dan Default Route

Dari sumber MikroTik Documentation (2024), gateway adalah perangkat jaringan yang bertindak sebagai pintu gerbang atau titik keluar-masuk antara dua jaringan yang berbeda, biasanya antara jaringan lokal (LAN) dengan jaringan luas (WAN) atau internet. Gateway memiliki peran penting dalam mengarahkan traffic jaringan yang tidak memiliki rute spesifik ke tujuan yang tepat.


Dari sumber NetworkWorld (2023), default route merupakan rute cadangan yang digunakan oleh router ketika tidak ada rute spesifik yang cocok untuk paket data yang akan diteruskan. Default route biasanya mengarah ke gateway yang terhubung ke internet atau jaringan yang lebih luas, sehingga semua traffic yang tidak memiliki rute khusus akan diarahkan melalui jalur ini.


Kesimpulan: Dari 2 pendapat tersebut dapat disimpulkan bahwa gateway dan default route merupakan komponen fundamental dalam infrastruktur jaringan yang memungkinkan komunikasi antar jaringan. Gateway berfungsi sebagai penghubung fisik, sedangkan default route adalah aturan logis yang menentukan kemana traffic harus diarahkan ketika tidak ada rute spesifik yang tersedia.

1. KONFIGURASI GATEWAY DASAR

Tampilan konfigurasi gateway pada MikroTik RouterOS

Dari sumber MikroTik Wiki (2024), untuk mengkonfigurasi gateway pada MikroTik, langkah pertama adalah menentukan interface yang akan berfungsi sebagai gateway, biasanya interface yang terhubung ke ISP atau router upstream. Gateway harus memiliki alamat IP yang dapat dijangkau dan berada dalam subnet yang sama dengan interface router.


Dari sumber RouterOS Manual (2023), setelah interface dikonfigurasi dengan IP address yang sesuai, gateway dapat diatur menggunakan perintah ip route untuk membuat default route yang mengarah ke alamat IP gateway ISP atau router upstream.

Berikut perintah CLI untuk mengatur IP address pada interface WAN:

/ip address add address=192.168.1.100/24 interface=ether1
# Menambahkan IP address pada interface ether1 (WAN)

Kemudian, konfigurasikan default route yang mengarah ke gateway ISP:

/ip route add dst-address=0.0.0.0/0 gateway=192.168.1.1
# Menambahkan default route ke gateway ISP

Untuk verifikasi konfigurasi gateway, gunakan perintah berikut:

/ip route print
# Melihat tabel routing yang aktif
0 ADC 192.168.1.0/24 0 ether1
1 ADC 0.0.0.0/0 1 192.168.1.1

Dari sumber MikroTik Documentation (2024), pastikan interface yang digunakan sebagai gateway memiliki status "running" dan "enabled". Jika interface down, maka default route tidak akan berfungsi dan traffic tidak akan dapat keluar dari jaringan lokal.

Untuk mengecek status interface:

/interface print
# Memeriksa status semua interface
0 R ether1 true 1500
1 ether2 true 1500
Kesimpulan: Dari 2 pendapat tersebut dapat disimpulkan bahwa konfigurasi gateway dasar memerlukan dua langkah utama: mengatur IP address pada interface WAN dan menambahkan default route yang mengarah ke gateway ISP. Proses ini memungkinkan router MikroTik untuk meneruskan traffic dari jaringan lokal ke internet melalui jalur yang telah ditentukan.

2. MENAMBAHKAN DEFAULT ROUTE

Konfigurasi default route melalui Winbox MikroTik

Dari sumber MikroTik Manual (2023), default route adalah entry dalam tabel routing dengan destination 0.0.0.0/0 yang akan menangkap semua traffic yang tidak memiliki rute spesifik. Default route memiliki prioritas terendah dalam tabel routing dan hanya akan digunakan jika tidak ada rute yang lebih spesifik tersedia untuk destination tertentu.


Dari sumber NetworkLessons (2024), dalam implementasinya, default route dapat dikonfigurasi dengan berbagai parameter tambahan seperti distance, routing-mark, dan check-gateway untuk meningkatkan fleksibilitas dan redundansi jaringan. Parameter distance digunakan untuk menentukan prioritas jika ada beberapa default route yang tersedia.

Perintah dasar untuk menambahkan default route:

/ip route add
dst-address=0.0.0.0/0
gateway=192.168.1.1
distance=1
# Menambahkan default route dengan distance 1

Untuk konfigurasi default route dengan check gateway:

/ip route add
dst-address=0.0.0.0/0
gateway=192.168.1.1
check-gateway=ping
distance=1
# Default route dengan monitoring gateway menggunakan ping

Berhati-hatilah saat menghapus default route yang sedang aktif. Menghapus default route tanpa memiliki rute alternatif dapat menyebabkan koneksi internet terputus. Pastikan selalu memiliki akses alternatif ke router sebelum melakukan perubahan pada routing table.

Untuk menambahkan default route dengan routing mark:

/ip route add
dst-address=0.0.0.0/0
gateway=192.168.1.1
routing-mark=main-route
distance=1
# Default route dengan routing mark untuk policy routing

Untuk melihat detail route yang aktif:

/ip route print detail
# Menampilkan informasi lengkap routing table
0 dst-address=0.0.0.0/0 gateway=192.168.1.1
distance=1 scope=30 target-scope=10
routing-mark="" check-gateway=ping
Kesimpulan: Dari 2 pendapat tersebut dapat disimpulkan bahwa default route merupakan komponen penting dalam tabel routing yang berfungsi sebagai jalur cadangan untuk semua traffic yang tidak memiliki rute spesifik. Dengan parameter tambahan seperti distance, check-gateway, dan routing-mark, default route dapat dikonfigurasi untuk memberikan redundansi dan fleksibilitas dalam manajemen traffic jaringan.

3. TROUBLESHOOTING GATEWAY ISSUES

Dari sumber MikroTik Troubleshooting Guide (2024), masalah gateway yang paling umum meliputi gateway unreachable, routing loop, dan konfigurasi IP address yang salah. Proses troubleshooting harus dilakukan secara sistematis mulai dari layer fisik hingga layer aplikasi untuk mengidentifikasi sumber masalah dengan akurat.


Dari sumber Network Troubleshooting Best Practices (2023), tools diagnostik bawaan MikroTik seperti ping, traceroute, dan torch sangat berguna untuk menganalisis masalah konektivitas. Kombinasi dari berbagai tools ini dapat memberikan gambaran lengkap tentang kondisi jaringan dan membantu lokalisasi masalah dengan cepat.

Perintah untuk mengecek konektivitas ke gateway:

/ping 192.168.1.1 count=5
# Test ping ke gateway untuk cek konektivitas
SEQ HOST SIZE TTL TIME STATUS
0 192.168.1.1 56 64 1ms
1 192.168.1.1 56 64 1ms

Traceroute untuk melihat jalur paket:

/tool traceroute 8.8.8.8
# Trace jalur paket ke DNS Google
1 192.168.1.1 1ms 2ms 1ms
2 10.10.10.1 5ms 4ms 6ms
3 8.8.8.8 15ms 14ms 16ms

Saat melakukan troubleshooting, hindari melakukan perubahan konfigurasi secara bersamaan di multiple area. Lakukan perubahan satu per satu dan test hasilnya untuk memastikan tidak menimbulkan masalah baru atau memperburuk kondisi yang sudah ada.

Mengecek ARP table untuk masalah Layer 2:

/ip arp print
# Cek ARP table untuk gateway
0 D 192.168.1.1 aa:bb:cc:dd:ee:ff ether1
1 192.168.1.5 11:22:33:44:55:66 ether1

Monitoring traffic dengan torch:

/tool torch interface=ether1 duration=10
# Monitor traffic real-time di interface WAN
TX: 145.2kbps (182 packets) RX: 1.2Mbps (1540 packets)

Reset routing table jika diperlukan:

/ip route remove [find dst-address=0.0.0.0/0]
# Menghapus semua default route
/ip route add dst-address=0.0.0.0/0 gateway=192.168.1.1
# Menambahkan kembali default route yang benar

Dari sumber MikroTik Support (2024), jika masalah persisten setelah troubleshooting dasar, periksa log system untuk error message yang dapat memberikan petunjuk lebih spesifik. Log dapat diakses melalui /log print atau melalui Winbox di menu Log.

Melihat system log untuk error:

/log print where topics~"route"
# Filter log untuk masalah routing
jan/01 12:34:56 route,info route added
jan/01 12:35:12 route,error gateway 192.168.1.1 unreachable
Kesimpulan: Dari 2 pendapat tersebut dapat disimpulkan bahwa troubleshooting gateway issues memerlukan pendekatan sistematis menggunakan berbagai tools diagnostik yang tersedia. Dengan kombinasi ping, traceroute, ARP table check, dan monitoring log, administrator dapat mengidentifikasi dan menyelesaikan masalah gateway dengan efisien dan tepat sasaran.
